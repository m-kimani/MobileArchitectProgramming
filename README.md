###Event Tracker Mobile Application###

  My Event Tracker app was designed to help users organize and manage their personal events in one place. The app allows users to create an account, securely log in, and add, edit, delete, and view events in a grid layout. I also included optional SMS notifications so users can receive reminders about their events. The app was designed to address the need of busy professionals and students for a simple and organized way to keep track of upcoming activities and appointments.

  To support user needs, I created several screens including a login screen, an event dashboard, an event editing screen, and an SMS settings screen. I focused on making the interface straightforward so users can easily navigate the application by using clearly labeled buttons, cards, and forms that help guide users through each feature. During development, I broke the project into smaller pieces by implementing one feature at a time and testing frequently in the Android Emulator. This helped me quickly identify issues with the database, RecyclerView, and activity navigation by tracking the errors being logged and fixing them along the way. Testing was important because it revealed bugs that prevented data from displaying correctly and helped ensure the app functioned as intended. One challenge I had to overcome was creating the relationship between users and their events while supporting CRUD database operations. I believe I was most successful in developing the database functionality and event management system because it demonstrates my ability to design a persistent database, build a user-centered interface, and connect multiple activities together into a functional application, which I've obtained experience in developing full-stack web applications.
